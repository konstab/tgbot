ШАГ 1. Подготовка на ТВОЁМ рабочем сервере
1.1. Сделать бэкап базы (ОБЯЗАТЕЛЬНО)
cd /home/botuser/tgbotnails
cp bot.db backups/bot_client_init_$(date +%F_%H-%M-%S).db


Если клиент новый (пустая база) — можешь просто взять bot.db как есть
Если клиент переносится — берёшь актуальный bot.db

1.2. Подготовить .env ДЛЯ КЛИЕНТА

На рабочем сервере НЕ копируй свой .env.

Создай временный файл:

nano /tmp/client.env


Пример:

BOT_TOKEN=ТОКЕН_КЛИЕНТА


(у каждого клиента свой токен — это важно)

ШАГ 2. Подключение к серверу клиента
ssh user@CLIENT_SERVER_IP

ШАГ 3. Забрать код с GitHub
cd ~
git clone https://github.com/konstab/tgbotnails.git
cd tgbotnails


(Если репо приватное — либо ты логинишься, либо используешь SSH-ключ)

ШАГ 4. Перенести настройки и данные
4.1. Закинуть .env

Самый простой способ — вставить руками:

nano .env


Вставляешь содержимое из /tmp/client.env

4.2. Закинуть базу данных

С твоего сервера:

scp /home/botuser/tgbotnails/bot.db user@CLIENT_SERVER_IP:~/tgbotnails/bot.db

ШАГ 5. Запуск бота
docker compose up -d --build
docker compose ps


Если Up — всё, бот жив.

ШАГ 6. Проверка
docker logs -f tgbotnails


Пишешь боту в Telegram → видишь активность → ✅

ЕСЛИ КЛИЕНТ СОВСЕМ НОВЫЙ (без базы)

Просто не копируешь bot.db.
SQLite создаст пустую базу автоматически при первом запуске.

ЕСЛИ НУЖНО “ОБНОВИТЬ” УЖЕ РАБОТАЮЩЕГО КЛИЕНТА
cd ~/tgbotnails
git pull
docker compose up -d --build

КРИТИЧЕСКИЕ ПРАВИЛА (ОЧЕНЬ ВАЖНО)

❗ Один клиент = один сервер = один .env
❗ Никогда не шарь токены
❗ Никогда не запускай двух ботов с одной БД
❗ Всегда делай бэкап bot.db перед обновлением
